Flagship Project

Heart Disease Detection System

A healthcare ML workflow that combines structured clinical inputs, multi-model experimentation, and a Flask-based screening experience designed to feel useful to a reviewer, not just technically correct in isolation.

Open Portfolio Demo Open Notebook File Download PDF
Algorithms tested 5
Interface stack Flask + HTML/CSS
Dataset proof heart_data_set.csv
Deliverables Notebook + model + web app

Problem

Build a screening flow that can take basic patient features such as age, chest pain type, cholesterol, blood pressure, and ECG-linked inputs, then produce a clear heart-disease prediction in a usable web interface.

Dataset

The project uses a structured tabular dataset stored locally as pngm/dataset/heart_data_set.csv with 13 input features and one target label for heart-disease prediction.

Model

Multiple classical ML algorithms were explored: Logistic Regression, SVM, Decision Tree, Random Forest, and KNN. The portfolio demo is currently presented around the Random Forest workflow because it fits the decision-support story best.

Pipeline

Clinical inputs are captured through the Flask form, cleaned and mapped into model features, passed through the trained estimator, and returned as a prediction/report-style result for the user.

Results

The strongest proof here is not just a score card. It is the combination of trained artifacts, notebook workflow, multiple algorithms, a saved model file, and a working Flask interface that produces printable output.

Future Scope

Add calibration metrics, SHAP-style explainability, patient-history support, doctor-facing dashboards, and secure hosted deployment for a more clinically credible screening assistant.

Limitations

  • This is a project-grade screening tool, not a medically certified diagnostic system.
  • The portfolio presentation does not yet expose model calibration, ROC curves, or validation summary in the UI.
  • The live demo is designed for local use and review, not clinical deployment.

Why It Is the Flagship

  • Strongest combination of data, ML experimentation, and a real Flask app.
  • Clear business and social relevance through healthcare screening.
  • Best showcase of end-to-end thinking: dataset, models, UI, and printable outcome.
