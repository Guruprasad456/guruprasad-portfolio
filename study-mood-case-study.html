Case Study 02

AI Study Mood Detector

A study-support concept built on an image dataset, mapped into broader focus states, then translated into a live coaching interface with camera readiness, task parsing, and recovery recommendations.

Problem

Go beyond static mood detection and create a product that can help students decide whether to focus, reset, simplify the task, or change the environment.

Dataset

The model uses the local image dataset stored under study-mood-app/study mood generator dataset, then maps seven emotional classes into four broader study states: focused, tired, distracted, and stressed.

Model

The saved metrics identify the model as a visual centroid ensemble using 20 x 20 images and a 1265-length feature representation.

Pipeline

Dataset images are resized, transformed into compact visual features, mapped from emotion labels into study-state labels, scored by the ensemble, and then used by the demo to generate intervention guidance.

Results

The current measured accuracy is 31.37% on the broad state mapping. That makes this a learning-forward productivity AI prototype, not a finished emotion-recognition product.

Future Scope

Add stronger CNN-based vision, temporal face-state tracking, user-specific calibration, attention markers, and more reliable environment fusion from audio and brightness signals.

Limitations

  • Accuracy remains modest, so the current version should be seen as a coaching prototype.
  • The state mapping compresses seven emotions into four study states, which improves usefulness but removes nuance.
  • The demo combines model-backed logic with product heuristics to stay practical in-browser.

Where It Can Grow

  • Real webcam inference with stronger face embeddings or CNN features.
  • Audio/noise fusion, historical focus tracking, and personalized study baselines.
  • Clearer evaluation on new student-specific or classroom-style datasets.
