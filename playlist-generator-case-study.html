Case Study 04

Mood-Based Playlist Generator

A music recommendation concept built from labelled audio features, semantic mood mapping, and a front-end mix builder that makes the recommendation logic feel like a real curation product.

Open Demo Open Training Script Download PDF

Problem

Move beyond simple genre choice and recommend a believable playlist based on mood, energy, lyric density, and session type.

Dataset

The model uses a local 278k labelled playlist dataset with audio features like danceability, energy, loudness, speechiness, acousticness, instrumentalness, valence, and tempo.

Model

The metrics describe the engine as an audio feature centroid classifier that maps tracks into focused, happy, energetic, and reflective labels.

Pipeline

Structured audio features are normalized into centroid-based mood labels, then translated into front-end recommendation lanes and exportable playlist actions.

Results

The saved metrics report 77.41% label accuracy, which makes the project strong enough for a portfolio-grade recommendation demo with real dataset proof.

Future Scope

Add platform authentication, playlist creation APIs, user listening history, collaborative filtering, and stronger multi-language mood detection from lyrics or metadata.

Before / After Workflow

How the project upgrades mood-based listening from guesswork to structured curation.

Before

Manual playlist searching

  • Users search by mood words without a structured feature route
  • Energy, lyric density, and session type are hard to balance together
  • The output is often a loose playlist instead of a curation flow

After

Feature-backed mix design

  • Audio features support a repeatable mood route
  • The demo turns prompt + mood + platform into a cleaner mix plan
  • Export and sharing actions make the recommendation feel more product-ready
