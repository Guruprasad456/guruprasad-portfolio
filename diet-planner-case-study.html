Case Study 03

AI-Powered Diet Planner

A nutrition planning assistant built around practical adherence: calorie fit, food preference match, recommendation clustering, and a product-style planning board that turns a dataset into a usable daily structure.

Problem

Translate goals, restrictions, cuisine preference, exercise, and adherence signals into a meal direction that feels practical enough to follow.

Dataset

The planner uses local nutrition data from daily_food_nutrition_dataset.csv and recommendation data from diet_recommendations_dataset.csv.

Model

The metrics file describes the engine as a nutrition vector recommender with a lightweight cluster-and-prototype approach, which fits the small recommendation-focused dataset.

Pipeline

Nutrition values and diet recommendations are organized into clusters, matched against user context, then turned into a daily plan with macros, grocery hints, and execution guidance.

Results

The saved metrics report an 85.71% preference match rate and a 0.6897 cluster separation score, giving the planner enough structure to support product-style planning outputs.

Future Scope

Add a larger food database, micronutrient optimization, user history, clinical nutrition constraints, budget tracking, and explainable swaps for long-term adherence.

Before / After Workflow

How the planner improves a vague meal goal into a structured daily direction.

Before

Generic diet advice only

  • Users often receive broad recommendations that are hard to follow
  • No planning structure tied to budget, prep time, or preference
  • Meal execution is disconnected from the recommendation itself

After

Recommendation becomes a planner

  • User intent is translated into a practical board-style plan
  • Cluster logic supports macros, swaps, and routine guidance
  • The output feels closer to a usable product than a static list

Why the model choice fits

  • The dataset is small and recommendation-oriented, so a lightweight prototype recommender is more honest than pretending to have a large deep model.
  • Clustering helps the UI explain meal direction and adherence trade-offs clearly.
  • The app behaves like a planner rather than a black-box predictor, which matches the user need.
